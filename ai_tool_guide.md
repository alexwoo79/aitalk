# AI 工具选择指南

> 不同 AI 工具各有所长，选对工具能让学习和工作效率翻倍

---

## 工具速查表

| 场景 | 推荐工具 | 理由 |
|------|---------|------|
| 解释编程概念 | ChatGPT / Claude | 长文本解释能力强，类比丰富 |
| 写代码/生成项目 | Claude / Copilot / QoderWork | 代码质量高，支持完整项目生成 |
| 实时写代码（IDE 内） | GitHub Copilot / Cursor | 嵌入编辑器，边写边补全 |
| 数据分析 | ChatGPT (Code Interpreter) | 可上传文件，直接运行并可视化 |
| 操作本地文件 | QoderWork | 直接读写本地文件，操作桌面应用 |
| 生成文档/PPT | QoderWork / ChatGPT | 可以直接生成 .docx/.pptx 文件 |
| 代码审查 | Claude / ChatGPT | 粘贴代码即可获得详细 review 意见 |
| 调试排错 | Claude / Copilot Chat | 描述报错信息，快速定位问题 |
| 学习新语言 | Claude / ChatGPT | 交互式教学，可以根据你的水平调整 |
| 日常写作 | 任何工具 | 邮件、报告、翻译均可 |

## 主流工具详细对比

### ChatGPT（OpenAI）

**擅长**：通用对话、概念解释、数据分析（上传文件后直接运行）

**特点**：
- Code Interpreter 模式可以直接上传 CSV/Excel，AI 帮你分析并生成图表
- 对话记忆好，可以在一个会话里持续迭代
- 支持图片识别（截图报错信息让它帮忙 debug）

**局限**：
- 不能直接操作你的本地文件
- 生成的代码需要你自己复制粘贴到本地运行
- 无法生成 .docx/.pptx 等复杂文件格式

**适合谁**：需要快速分析数据、学习新概念、写日常文档的同事

---

### Claude（Anthropic）

**擅长**：长文本处理、代码生成、深度分析

**特点**：
- 支持超长上下文（可以一次喂入整个代码库）
- 代码生成质量高，结构清晰
- 分析能力深入，适合 Code Review 和架构设计讨论
- Artifacts 功能可以实时预览生成的 HTML/React 代码

**局限**：
- 不能运行代码，只能生成代码让你自己跑
- 不能直接操作本地文件

**适合谁**：需要做代码审查、架构设计、长文档分析的技术同事

---

### GitHub Copilot

**擅长**：IDE 内实时代码补全

**特点**：
- 直接嵌入 VS Code / JetBrains 等编辑器
- 写注释就能自动生成代码
- 支持 Chat 模式，选中代码后可以让 AI 解释/重构
- 对项目上下文的理解好（可以读取当前打开的文件）

**局限**：
- 主要在代码场景，通用写作能力弱
- 需要在编辑器里使用，不适合非技术人员

**适合谁**：日常在 IDE 里写代码的开发人员

---

### Cursor

**擅长**：AI 原生代码编辑器

**特点**：
- 基于 VS Code，学习成本低
- AI 可以直接编辑项目中的文件
- 支持引用项目文件作为上下文
- Composer 模式可以一次性修改多个文件

**局限**：
- 需要切换到 Cursor 编辑器
- 主要面向开发者

**适合谁**：希望 AI 深度参与编码过程的开发人员

---

### QoderWork

**擅长**：本地文件操作、桌面应用自动化、一站式办公

**特点**：
- 直接读写本地文件系统（不需要复制粘贴）
- 可以操作 macOS 原生应用（日历、提醒、邮件、备忘录等）
- 直接生成 Word/PPT/PDF/Excel 等格式文件
- 支持定时任务和自动化工作流
- 可以通过 MCP 连接 Slack、Asana 等第三方工具

**局限**：
- 需要安装桌面应用
- 某些操作需要用户授权

**适合谁**：需要 AI 深度参与日常工作流、处理本地文件的各类同事

## 组合使用建议

实际工作中，最有效的做法是**组合使用**多个工具：

**场景一：学习一门新编程语言**

1. 用 ChatGPT/Claude 解释概念，用生活类比帮你理解
2. 用 Copilot/Cursor 在 IDE 里边写边学
3. 遇到报错时截图发给 ChatGPT 帮忙 debug

**场景二：做月度数据分析**

1. 用 ChatGPT Code Interpreter 上传 Excel 快速探索数据
2. 确定分析方向后，让 Claude 生成正式的分析脚本
3. 用 QoderWork 直接生成格式化的报告文档

**场景三：开发一个小工具**

1. 用 Claude 讨论需求和架构设计
2. 用 Cursor/Copilot 实现代码
3. 用 Claude 做 Code Review
4. 用 QoderWork 辅助生成文档和演示

## 提示词写作通用技巧

无论使用哪个工具，好的提示词都遵循以下原则：

**1. 给角色**：告诉 AI 它是谁（"你是 Python 教学专家"）

**2. 给上下文**：说明背景（"我是业务分析师，不太懂编程"）

**3. 给具体任务**：越具体越好（"帮我写一个函数，输入 CSV 路径，输出按区域汇总的 DataFrame"）

**4. 给约束**：限定范围（"代码 30 行以内，注释用中文，不需要异常处理"）

**5. 给示例**：如果可能，给一个输入输出的例子

**6. 迭代追问**：第一轮结果不满意就追问（"太复杂了，简化一下"、"能换个例子吗"）

## 给初学者的建议

如果你之前没用过 AI 工具，建议这样起步：

**第一周**：用 ChatGPT 问问题。把它当一个永远有耐心的老师，问任何你不好意思问别人的问题。

**第二周**：让 AI 帮你写东西。邮件、会议纪要、简单的数据处理脚本——从你日常最重复的工作开始。

**第三周**：尝试用 AI 学习一个新技能。比如让它教你 pandas 基础操作，或者教你写一个简单的网页。

**第四周**：开始组合使用工具。把 AI 融入你的日常工作流，让它成为你的常规工具之一。
