# 计算列组合条件筛选 — 测试验证指南

## 测试数据

文件：`test_data/computed_col_filter_test.csv`

24 行销售数据，包含列：`日期, 区域, 产品类别, 销售额, 成本, 销量`

---

## 测试案例 1：变量独立筛选

### 计算列：`电子产品毛利`

| 字段 | 值 |
|------|-----|
| 变量 A | 列=销售额, 筛选=`产品类别 = 电子产品` |
| 变量 B | 列=成本, 无筛选 |
| 表达式 | `A - B` |
| 共享筛选 | 无 |

### 预期行为
- 电子产品行：A=销售额, B=成本 → `销售额 - 成本` 正常计算
- 家居用品行：A=0（不满足筛选）, B=成本 → `0 - 成本` 结果为负数

### 预期结果（前 3 行）

| 产品类别 | 销售额 | 成本 | A(筛选后) | B | 电子产品毛利 |
|---------|--------|------|----------|---|------------|
| 电子产品 | 158000 | 120000 | 158000 | 120000 | **38000** |
| 家居用品 | 45000 | 35000 | 0 | 35000 | **-35000** |
| 电子产品 | 98000 | 75000 | 98000 | 75000 | **23000** |

## 测试案例 3：数值条件筛选

### 计算列：`高利润销售额`

| 字段 | 值 |
|------|-----|
| 变量 A | 列=销售额, 筛选=`销售额 > 100000` |
| 变量 B | 列=成本, 筛选=`销售额 > 100000` |
| 表达式 | `A - B` |
| 共享筛选 | 无 |

### 预期行为
- 销售额>100000：正常计算 A-B

### 预期结果（部分行）

| 销售额 | 成本 | 高利润销售额 |
|--------|------|------------|
| 158000 | 120000 | **38000** |
| 45000 | 35000 | **0** |
| 185000 | 140000 | **45000** |
| 38000 | 30000 | **0** |

## 测试案例 4：复合表达式 + 多筛选

### 计算列：`电子产品利润率`

| 字段 | 值 |
|------|-----|
| 变量 A | 列=销售额, 筛选=`产品类别 = 电子产品` |
| 变量 B | 列=成本, 筛选=`产品类别 = 电子产品` |
| 变量 C | 列=销量, 筛选=`产品类别 = 电子产品` |
| 表达式 | `(A - B) / A * 100` |
| 共享筛选 | `区域 = 华东` |

### 预期结果

| 区域 | 产品类别 | 销售额 | 成本 | 利润率 |
|------|---------|--------|------|--------|
| 华东 | 电子产品 | 158000 | 120000 | **24.05** |
| 华东 | 家居用品 | 45000 | 35000 | **0** |
| 华北 | 电子产品 | 185000 | 140000 | **0** (共享筛选不匹配) |

## 验证步骤

1. 上传 `test_data/computed_col_filter_test.csv`
2. 进入配置页 → 2.1 数据 & 指标 tab
3. 在计算列区域按上述参数依次添加 4 个计算列
4. 切换到 2.2 配置 & 预览 tab，点击生成 Dashboard
5. 在 Dashboard 表格中查看计算列的值是否与预期一致

## 关键验证点

- [ ] 变量筛选：不满足条件的变量值为 0
- [ ] 共享筛选：不满足共享筛选的行结果为 0
- [ ] 数值条件：`>`, `<`, `>=`, `<=` 正确生效
- [ ] 字符串条件：`=`, `!=` 正确生效
- [ ] JS 端预览：ConfigView 右侧预览表格计算列值正确
- [ ] Rust 端：Dashboard 页面表格/图表中计算列值正确
