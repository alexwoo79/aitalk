# SmartBoard 筛选条件语法 — 培训手册

> 数据源：`test_data/time_analysis_sample.csv`（1296行，4区域×3渠道×3产品线×36个月）

---

## 一、运算符速查表

| 运算符 | 含义 | 适用列类型 | 示例 |
|--------|------|-----------|------|
| `=` | 等于 | 全部 | `region = 华东` |
| `!=` | 不等于 | 全部 | `channel != 代理商` |
| `>` | 大于 | 数值列 | `revenue > 100000` |
| `<` | 小于 | 数值列 | `orders < 300` |
| `>=` | 大于等于 | 数值列 | `revenue >= 80000` |
| `<=` | 小于等于 | 数值列 | `returns <= 5` |
| `in` | 值在列表中 | 文本列 | `region in 华东,华南` |
| `~` 或 `～` | 包含子串 | 文本列 | `product_line ~ A` |
| `contains` | 同 `~` | 文本列 | `channel contains 线` |

---

## 二、逻辑连接符

| 符号 | 含义 | 优先级 |
|------|------|--------|
| `&` | AND（且）| 先拆分 |
| `|` | OR（或）| 后拆分 |

**解析规则**：先按 `&` 拆分为 AND 组，每个 AND 组内按 `|` 拆分为 OR 条件。

```
A = 1 | A = 2 & B = 3
→ AND组1: A = 1 | A = 2    （OR：满足任一）
→ AND组2: B = 3             （AND：必须全部满足）
→ 最终：(A=1 或 A=2) 且 B=3
```

---

## 三、兼容性特性

| 特性 | 说明 | 示例 |
|------|------|------|
| 全角 `～` | 中文输入法的 `～` 等同于 `~` | `product_line ～ B` = `product_line ~ B` |
| 大小写列名 | 列名不区分大小写 | `REGION = 华东` = `region = 华东` |
| 列表分隔符 | `in` 支持 `,` `，` `、` `;` 空格 | `region in 华东,华南,华北` |
| 列表括号 | `in` 值支持 `[]` `()` 包裹 | `region in [华南,华东]` |

## 五、常见错误对照

| ❌ 错误写法 | 问题 | ✅ 正确写法 |
|------------|------|------------|
| `region = 华东,华南` | 等值只能一个值 | `region in 华东,华南` |
| `revenue ~ 100` | 数值列不宜用 contains | `revenue = 100` 或 `revenue > 100` |
| `region in 华东 & 华南` | `&` 是 AND 不是列表分隔 | `region in 华东,华南` |
| `(region = 华东 \| region = 华南) & channel = 线上` | 不支持括号 | `region = 华东 \| region = 华南 & channel = 线上` |
| `product_line contains A`（中文输入法）| 全角英文可能不识别 | 用 `~` 或 `～`，不用 `contains` 也行 |
