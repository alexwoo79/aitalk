# KPI 公式引擎计算逻辑说明

> 版本：2026-06-26 | 模块：`src/core/formula-engine.ts`

---

## 1. 公式语法

### 1.1 变量定义

每个变量用一个大写字母作为别名，映射到数据文件中的实际列名：

```
A = 单价
B = 数量
C = 折扣
```

别名按添加顺序自动分配：A、B、C……Z，超过 26 个后依次为 AA、AB……

### 1.2 表达式

聚合函数写入表达式内部，引擎自动判断计算模式：

| 表达式 | 计算模式 | 等价语义 |
|--------|---------|---------|
| `SUM(A*B)` | 行内计算 → 聚合 | 逐行计算 单价×数量，再求和 |
| `SUM(A)+SUM(B)` | 先聚合 → 再计算 | 先分别求 SUM(单价) 和 SUM(数量)，再相加 |
| `SUM(A*B)/SUM(B)*100` | 混合模式 | 行内计算 A×B 后 SUM，除以 SUM(B)，乘 100 |
| `AVG(A)` | 直接聚合 | 求 A 列均值 |
| `COUNT(A)` | 计数 | 统计 A 列非空行数 |
| `UNIQUE_COUNT(A)` | 唯一计数 | 统计 A 列去重值个数 |
| `MIN(A*B)` | 行内 → MIN | 逐行 A×B，取最小值 |
| `MAX(A*B)` | 行内 → MAX | 逐行 A×B，取最大值 |

## 2. 计算流程

```
                    输入
                     │
    ┌────────────────┼────────────────┐
    │  variables     │  expression    │  filter(共享)
    │  A=单价 B=数量  │ SUM(A*B)/SUM(B) │  区域=华东
    └────────────────┼────────────────┘
                     │
                     ▼
            ┌─────────────────┐
            │ 1. 应用共享筛选   │  filter ≠ 空 → 先过滤行
            │    剩余 60 行     │
            └────────┬────────┘
                     │
                     ▼
            ┌─────────────────────┐
            │ 2. 解析表达式        │  找到 SUM(A*B) 和 SUM(B)
            │    每个聚合调用 = 一个 term │
            └────────┬────────────┘
                     │
        ┌────────────┴────────────┐
        ▼                         ▼
  ┌──────────────┐         ┌──────────────┐
  │ term #1      │         │ term #2      │
  │ SUM(A*B)     │         │ SUM(B)       │
  │ 涉及变量: A,B │         │ 涉及变量: B   │
  │              │         │              │
  │ 叠加筛选:    │         │ 叠加筛选:    │
  │ 共享 & A筛选 │         │ 共享 & B筛选 │
  │ & B筛选      │         │              │
  │ → 剩余 40 行 │         │ → 剩余 55 行 │
  │              │         │              │
  │ 逐行计算:    │         │ 逐行取值:    │
  │ 行1: 10×5=50 │         │ 行1: 5      │
  │ 行2: 20×3=60 │         │ 行2: 3      │
  │ ...          │         │ ...          │
  │              │         │              │
  │ SUM → 1400   │         │ SUM → 80    │
  └──────┬───────┘         └──────┬───────┘
         │                        │
         └────────┬───────────────┘
                  ▼
         ┌────────────────┐
         │ 3. 替换回表达式  │  1400 / 80 * 100
         │    计算结果      │  = 1750
         └────────────────┘
```

## 3. 筛选条件机制

公式支持三个层级的筛选条件：

```
┌──────────────────────────────────────────────┐
│              共享筛选 (formula.filter)         │
│         对所有聚合调用生效，先于其他筛选执行       │
│  ┌─────────────────────────────────────────┐ │
│  │        变量 A 独立筛选 (A.filter)         │ │
│  │  仅当 A 出现在聚合函数内时叠加             │ │
│  │  ┌────────────────────────────────────┐ │ │
│  │  │    变量 B 独立筛选 (B.filter)       │ │ │
│  │  │  仅当 B 出现在聚合函数内时叠加       │ │ │
│  │  └────────────────────────────────────┘ │ │
│  └─────────────────────────────────────────┘ │
└──────────────────────────────────────────────┘
```

### 3.1 组合方式

- 所有筛选用 **&（AND）** 连接
- 即：共享 AND A筛选 AND B筛选 → 全部满足才参与计算

### 3.2 作用范围

```
SUM(A*B)     → 叠加 共享 & A筛选 & B筛选
SUM(A)       → 叠加 共享 & A筛选
SUM(B)       → 叠加 共享 & B筛选
SUM(A)+SUM(B) → 两个 term 各自独立叠加筛选，互不影响
```

### 3.3 不满足条件的处理

**整行排除**，不是当 0 代入。

| 聚合函数 | 排除 vs 置零的区别 |
|---------|-------------------|
| SUM | 等效（排除 = 不累加 = 贡献 0） |
| AVG | **不同！** 排除减少分母，置零拉低均值 |
| COUNT | **不同！** 排除减少计数 |
| MIN/MAX | **不同！** 排除可能改变极值 |

### 3.4 筛选语法

格式：`列名 运算符 值`

| 运算符 | 含义 | 示例 |
|--------|------|------|
| `=` | 等于 | `区域 = 华东` |
| `!=` | 不等于 | `金额 != 0` |
| `>` `<` `>=` `<=` | 大小比较 | `单价 > 10` |
| `~` `～` `contains` | 包含子串 | `项目名 ~ 设计` |
| `in` | 值在列表中 | `区域 in [华东, 华北]` |

复合条件：`&` = AND，`|` = OR。例如 `区域 = 华东 & 金额 > 100`。

## 4. 数据示例验证

### 输入数据

| 区域 | 单价 | 数量 |
|------|------|------|
| 华东 | 10 | 5 |
| 华东 | 20 | 3 |
| 华北 | 30 | 2 |

### 测试用例

| 表达式 | 筛选 | 计算过程 | 结果 |
|--------|------|---------|------|
| `SUM(A*B)` | 无 | 10×5+20×3+30×2 | **140** |
| `SUM(A)+SUM(B)` | 无 | (10+20+30)+(5+3+2) | **70** |
| `SUM(A*B)` | `区域 = 华东` | 10×5+20×3 | **110** |
| `SUM(A*B)` | A筛选 `单价 > 10` | 20×3+30×2 | **120** |
| `AVG(A)` | `区域 = 华东` | (10+20)/2 | **15** |
| `SUM(A*B)/SUM(B)*100` | 无 | 140/10×100 | **1400** |
| `COUNT(A)` | 无 | 3 行非空 | **3** |

## 5. 保存参数（可复用公式）

保存公式 KPI 后，其标签出现在其他公式的列选择器中（🔢 前缀）。

- 引用时**实时重新计算**
- 支持**依赖链**：KPI-A 引用 KPI-B，KPI-B 引用 KPI-C
- 最大深度 **5 层**防止循环引用

```
KPI-客单价 = SUM(A*B) / SUM(B)   → 保存标签 "客单价"
KPI-利润率 = SUM(C) / SUM(A*B)    → 在 A 列选择器中选 "🔢 客单价"
                                  → 引用时先计算 "客单价"，再代入
```

## 6. 模块结构

```
src/core/formula-engine.ts    ← 表达式解析 + 自适应计算
    ├── parseExpression()      ← 提取 SUM/AVG/COUNT... 调用
    ├── computeRowWise()       ← 逐行替换 A/B/C 为原始值并计算
    ├── applyAgg()             ← SUM/AVG/COUNT/MIN/MAX/UNIQUE_COUNT
    └── computeFormula()       ← 主入口

src/stores/preview-store.ts
    └── computeKpiValue()      ← 集成 formula-engine，处理 🔢 保存参数

src/views/ConfigView.vue
    └── KPI 公式编辑器 UI       ← 变量别名 + 筛选条件 + 表达式输入
```
