# 纯 TS 架构 → Rust 融合架构 — 提升分析

> 日期：2026-06-27  
> 项目：SmartBoard 智能数据分析看板  
> 对比基线：纯 TypeScript（浏览器端 SheetJS + PapaParse + JS 聚合）  
> 当前架构：Tauri + Rust/Polars 后端 + TypeScript 前端双引擎

---

## 一、架构对比

```
【纯 TS 架构】                          【当前 Rust 融合架构】

 CSV/Excel 解析                          CSV/Excel 解析
   ├─ SheetJS (JS, 单线程)                ├─ calamine (Rust, 原生)
   └─ PapaParse (JS, 单线程)              └─ Polars CSV reader (SIMD)
        │                                      │
        ▼                                      ▼
 JS 内存数组                             Polars DataFrame
 (V8 heap, GC 压力大)                    (Arrow 列式存储, 零拷贝)
        │                                      │
        ▼                                      ▼
 JS 聚合/筛选/公式                       Rust 端: Polars groupby/join
 (for-loop, reduce, filter)              TS 端:  简单聚合回退
```

### 双引擎调度

```
                    ┌─────────────────────┐
                    │   use-rust-bridge   │
                    │   (isTauri 检测)     │
                    └──────┬──────────────┘
                           │
           ┌───────────────┴───────────────┐
           ▼                               ▼
   Tauri 桌面环境                    浏览器环境

```

## 二、性能提升（量化估算）

| 操作 | 纯 TS | Rust/Polars | 提升倍数 | 说明 |
|------|-------|-------------|----------|------|
| **10 万行 CSV 加载** | ~800ms | ~80ms | **~10×** | Polars 多线程 + SIMD 解析 |
| **两表 JOIN（5 万 × 3 万）** | ~2000ms (O(n×m)) | ~50ms (hash join) | **~40×** | `preview-store.ts` 嵌套循环 → Polars 原生 hash join |
| **分组聚合（10 万行 × 3 维度）** | ~300ms | ~15ms | **~20×** | JS reduce → Polars lazy groupby |
| **内存占用（100 万行）** | ~600MB (V8) | ~120MB (Arrow) | **~5×** | Arrow 列式压缩 vs JS 对象开销 |
| **Excel 日期解析** | ❌ 错误输出 `45309` | ✅ 正确输出 `2024-01-18` | ∞ | 从 bug 到正确 |

> 注：TS 端数值为经验估算，Rust 端基于 Polars 官方基准和同类项目数据。实际提升取决于数据规模、列数、硬件配置。

## 四、架构优势

### 4.1 Rust 后端核心能力

| 模块 | 文件 | 核心能力 |
|------|------|----------|
| 文件加载 | `loader.rs` | CSV/Excel → Polars DataFrame，含日期序列号转换 |
| Excel 选区 | `excel_selection.rs` | 多 sheet 列举与按索引加载 |
| 数据合并 | `merge.rs` | JOIN（inner/left/right/outer）+ CONCAT（纵向拼接） |
| 分组聚合 | `groupby.rs` | sum/mean/count/min/max，Polars lazy API |
| 全局状态 | `state.rs` | `GLOBAL_DF` + `DATASET_REGISTRY` + `CLEAN_HISTORY` |

### 4.2 前端核心模块

| 模块 | 文件 | 核心能力 |
|------|------|----------|
| 数据存储 | `data-store.ts` | 多表状态、`effectiveHeaders`、分类查询 |
| 配置存储 | `config-store.ts` | Dashboard 配置、自动生成、快照/保存 |
| 预览存储 | `preview-store.ts` | 筛选应用、KPI 计算、spec 构建 |
| 桥接层 | `use-rust-bridge.ts` | Tauri invoke 封装、环境检测、浏览器降级 |
| 解析器 | `parser.ts` | CSV/XLSX 解析（浏览器回退） |
| 分类器 | `classifier.ts` | 列类型/角色自动推断、关联推荐 |

## 五、本次修改总结

```
P0（数据正确性）
  ├─ Excel 日期：从"输出错误数字 45309"到"输出正确日期 2024-01-18"
  └─ 列同步：Config ↔ Dashboard 实时一致，消除跨表列不一致

P1（跨表功能）
  ├─ 多表 JOIN：从 JS O(n×m) 到 Rust hash join（性能 ~40×）
  ├─ 配置页跨表：6 个区块全部可访问关联表列
  └─ 关联编辑：确认后可原地修改（新增 ✎ 编辑按钮）

P2（UI 一致性）
  └─ 表格列类型：跨表列正确显示 numeric / categorical / date
```

**一句话总结：**

> 从「单表单引擎 JS 原型」升级为「多表双引擎（Rust 高性能 + TS 全功能回退）的桌面级数据分析工具」。Rust/Polars 处理核心数据管线（加载/JOIN/聚合），TypeScript 负责 UI 交互、配置管理和浏览器降级。通过 `use-rust-bridge.ts` 统一调度，架构上已具备处理百万级数据的扩展能力。
