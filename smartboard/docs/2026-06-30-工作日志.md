# 2026-06-30 工作日志

## 一、计算列功能完善与修复

### 1.1 默认配置修复
- **移除保存时的自动汇总**：`saveComputedCol()` 不再自动添加 `summaryAggs[name] = 'sum'`
- **汇总行兜底移除**：`formatSummaryValue` 不再默认 `|| 'sum'`，无配置时返回 `—`
- **修复 Config 页预览**：`previewTableRows` 补上 `precomputeAggregates` 调用，含 `SUM()` 的计算列不再显示 `—`

### 1.2 浮点精度修复
- 根因：`A/SUM(A)` 合计 = 1.0000000000000002，`<= 1` 判断为 `false`，百分比显示 `1.00%` 而非 `100.00%`
- 修复：`fmtByChart` 和 `formatKpiValue` 的 `percent` 判断加上 $10^{-6}$ 容差

### 1.3 自动预设「占比」计算列
- `generateAutoTable()` 自动创建 `A/SUM(A)` 计算列，A 映射到推断的主指标
- i18n：中文「占比」、英文「Ratio」
- 自动附带 `metricDefaults`（percent 格式，table 分区）和 `summaryAggs`（求和）
- 去重：已有同名列不重复创建

### 1.4 占比逻辑提示
- 数据表上方蓝色提示条，计算列含聚合函数时自动显示
- 无顶部筛选：「子集占全局比，合计可能 < 100%」
- 有顶部筛选：「子集内相对占比，合计 = 100%」
- 中文/英文双语支持
- 培训文档：`docs/计算列占比逻辑说明.md`
