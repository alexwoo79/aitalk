## 二、URL 导入支持 Windows UNC 网络路径

**问题**：用户在 URL 导入输入 `\\172.16.0.209\03-创作中心（临时）\test_data\smartboard-sample-data.csv`，报 "请求异常: builder error"。

**根因**：
1. Windows 资源管理器复制路径时附带不可见 Unicode 字符（U+202A），导致 `is_local_path` 的 `starts_with("\\\\")` 匹配失败
2. 路径走到 HTTP 请求分支，reqwest 无法解析 → `builder error`

**修复**（4 个文件）：

| 文件 | 改动 |
|------|------|
| `src/components/upload/UrlImport.vue` | `stripInvisible()` 剥离 Unicode 控制字符；`translateRustError()` 翻译 Rust 错误 |
| `src-tauri/src/commands/datasource.rs` | `strip_invisible_chars()` + 路径归一化：保留 UNC 的 `\\` 前缀，仅将后续 `\` 转为 `/` |
| `src-tauri/src/commands/loader.rs` | 错误消息英文化 `File not found` |
| `src-tauri/src/commands/excel_selection.rs` | 同上 |
| `src/i18n/zh-CN.ts` / `en-US.ts` | 新增 `fileNotFound`、`dbFileNotFound`、`uncPathHint` |

## 四、ECharts Windows 高DPI 拉伸问题（多轮尝试）

**问题**：Windows 2880×2000 分辨率 + 250% 缩放 → 图表持续向下拉伸。

**尝试过程**：
1. 添加 `contain: layout style` → 导致双击全屏失效（`contain` 改变 `position: fixed` 的包含块）
2. 移除 `autoresize` + 手动 ResizeObserver → 退出全屏后同行图表高度错乱
3. 手动 ResizeObserver + `Math.floor` 抛光 → 全屏 ESC 后仍有副作用

**最终方案**：回到原始架构——`autoresize` + 纯全屏逻辑 + `.chart-card` 视觉兜底（`overflow: hidden` + `max-height: 80vh`）。全屏功能正常，Windows 拉伸由 CSS 裁剪兜底。

## 五、表格渲染方案评估

评估了引入 Element Plus `el-table-v2` 的建议文档（`docs/表格加载库的替换，提升显示效率.md`）。

**结论**：不引入。当前渐进加载（200 行 + IntersectionObserver 滚动追加）已能满足使用场景，`el-table-v2` 引入成本高且会破坏现有功能（条件行色、可变行高、汇总行等）。

## 六、Makefile 跨平台问题

macOS 执行 `make release-arm64` 失败——Makefile 大量使用 PowerShell、`findstr`、`[char]27` 等 Windows 命令。

**解决方案**：在 macOS 上直接使用 `npm run tauri build -- --target aarch64-apple-darwin`。
