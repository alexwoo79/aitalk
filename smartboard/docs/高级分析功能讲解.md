> 代码位置：`src/core/analysis.ts`
> 三个分析模块：时间序列 / 十分位 / K-means 聚类

---

## 一、时序分析（Timeseries）

### 算法原理

```
原始数据（逐行）
    │
    ▼ 按周期聚合（月/季/年）求和
    │
    ├─ 1️⃣ 原始值曲线
    ├─ 2️⃣ MA3 移动平均（3期平滑）
    ├─ 3️⃣ MoM 环比（较上期变化%）
    ├─ 4️⃣ YoY 同比（较去年同期变化%）
    ├─ 5️⃣ 线性回归趋势线
    └─ 6️⃣ 外推预测（未来3期）
```

### 各项指标详解

| 指标 | 公式 | 含义 |
|------|------|------|
| **原始值** | Σ 每期所有行的指标 | 该周期的总营收/总订单等 |
| **MA3** | `(本期 + 上期 + 上上期) ÷ 3` | 消除短期波动，看中期趋势 |
| **环比 MoM%** | `(本期 - 上期) ÷ |上期| × 100` | 较上月/上季增长多少 |
| **同比 YoY%** | `(本期 - 去年同月) ÷ |去年同月| × 100` | 消除季节因素的真实增长 |
| **趋势线** | 最小二乘法线性回归 | 长期走势方向 |
| **预测** | 趋势线外推 3 期 | 如果趋势持续会怎样 |

### 数据分析意义

| 分析目标 | 看什么 | 怎么解读 |
|---------|--------|---------|
| **发现季节性** | 原始值 + YoY | 每年同样的月份是否规律波动 |
| **判断增长** | 趋势线斜率 | 斜率为正=增长，为负=下滑 |
| **识别异常** | 环比 MoM% | 某月环比突然 >30% 或 < -20%，值得调查 |
| **平滑噪声** | MA3 vs 原始值 | MA3 更平滑，看清大方向 |
| **做预算** | 预测值 | 基于历史趋势预估未来 3 期 |

### 当前数据示例

```
选择: 日期=date, 指标=revenue, 周期=月
→ 36 个月度数据点（2023-01 ~ 2025-12）
→ 每年 12 月冲高（季节高峰），1-2 月回落（淡季）
→ 趋势线逐年上升（年增长约 12%）
→ 预测 2026 年 1-3 月营收
```

## 二、十分位分析（Decile）

### 算法原理

```
全部数据按指标排序（如 revenue 从低到高）
    │
    ▼ 等分为 10 个桶（每组数据量相同）
    │
D1 ─── D2 ─── D3 ─── ... ─── D10
最低                            最高
    │
    ▼ 每桶统计: 条数 | 总和 | 均值 | 范围
```

### 为什么是"十分位"而不是"分组"？

| 方法 | 分组方式 | 适用场景 |
|------|---------|---------|
| 等距分组 | 按数值区间 (0-1万, 1-2万...) | 分布均匀的数据 |
| **十分位** | 按数据量均分，每组 10% | 分布不均、有极端值的数据 |

十分位的优势：**每组样本量相同**，可以直接比较各组贡献。

### 数据分析意义

| 分析目标 | 怎么看 | 商业问题 |
|---------|--------|---------|
| **集中度分析** | D10 的总和 ÷ 全部总和 | 头部 10% 贡献了多少营收？ |
| **长尾效应** | D1~D5 的累计占比 | 底部 50% 的客户/产品是否值得维护？ |
| **贫富差距** | D10 均值 ÷ D1 均值 | 最高 10% 是最低 10% 的多少倍？ |
| **阈值设定** | D8/D9 的数值边界 | Top 20% 的起点在哪里？ |
| **异常检测** | D10 范围是否异常宽 | 是否存在极端离群值？ |

### 经典解读：帕累托法则（二八定律）

```
如果 D9+D10（前20%）的营收占比 > 80%
→ 符合二八定律，头部效应显著
→ 建议：重点维护头部客户/产品

如果 D1~D5（后50%）占比极低
→ 长尾可能不值得投入
→ 建议：考虑精简低效产品线
```

### 当前数据示例

```
选择: 指标=revenue
→ 1296 条数据排序，每组约 130 条
→ D1: 最低营收区间（小型代理商+C产品线）
→ D10: 最高营收区间（华东线上A产品线，12月旺季）
→ 可计算：D10 的营收总和是否远超 D1
```

## 三、聚类分析（K-means）

### 算法原理

```
选两个指标作为 X 轴和 Y 轴
    │
    ▼
┌──────────────────────────────────┐
│ 1. 初始化: 等间距选 K 个中心点    │
│ 2. 分配: 每个点归到最近的中心     │
│ 3. 更新: 重新计算每个簇的中心     │
│ 4. 重复 2-3，最多 20 轮或收敛    │
└──────────────────────────────────┘
    │
    ▼
输出: 散点图（不同颜色=不同簇）+ 中心点
```

### 关键参数

| 参数 | 默认值 | 含义 |
|------|-------|------|
| **K** | 3 | 聚成几类 |
| **X 轴指标** | 第 1 个指标列 | 散点图横轴 |
| **Y 轴指标** | 第 2 个指标列 | 散点图纵轴 |
| **迭代次数** | 最多 20 轮 | 自动收敛则提前停止 |

### 数据分析意义

| 分析目标 | 选什么指标 | 会发现什么 |
|---------|-----------|-----------|
| **客户分层** | 消费金额 × 购买频次 | 高价值/中等/低价值客户群 |
| **产品组合** | 营收 × 利润率 | 明星产品/现金牛/瘦狗产品 |
| **渠道效率** | 广告花费 × 营收 | 高ROI/中ROI/低ROI渠道 |
| **异常检测** | 任意两个指标 | 远离所有簇的孤立点=异常 |
| **市场细分** | 订单量 × 客单价 | 走量型/高客单型/均衡型 |

### 如何选择 K 值？

| K 值 | 适用场景 |
|------|---------|
| K=2 | 简单二分：高/低，好/坏 |
| K=3 | 经典三分：高/中/低（默认推荐） |
| K=4~5 | 更细粒度细分，需要业务经验判断 |

### 聚类结果的商业行动

```
簇1（高营收 + 高利润率）→ 明星产品 → 加大投入
簇2（高营收 + 低利润率）→ 走量产品 → 优化成本
簇3（低营收 + 低利润率）→ 瘦狗产品 → 考虑淘汰
```

### 当前数据示例

```
选择: X=revenue, Y=cost, K=3
→ 每个点 = 一条数据记录（某月某区域某渠道某产品线）
→ 簇1: 高营收高成本（华东线上A产品线旺季）
→ 簇2: 中等水平（大部分数据）
→ 簇3: 低营收低成本（华中代理商C产品线）
```

## 四、三者的关系与配合

```
                    ┌──────────────┐
                    │   原始数据    │
                    └──────┬───────┘
                           │
          ┌────────────────┼────────────────┐
          ▼                ▼                ▼
    ┌──────────┐    ┌──────────┐    ┌──────────┐
    │ 时间序列  │    │ 十分位   │    │  聚类    │
    │          │    │          │    │          │
    │ 看趋势    │    │ 看分布   │    │ 看结构   │
    │ WHEN     │    │ HOW MUCH │    │ WHO/WHAT│
    └──────────┘    └──────────┘    └──────────┘
          │                │                │
          ▼                ▼                ▼
    "什么时候好？"   "差距有多大？"   "谁和谁是一类？"
```

### 配合使用示例

```
场景：季度经营分析

1. 时间序列 → 发现 Q4 营收飙升，Q1 回落
   → 结论：季节性明显，Q4 是旺季

2. 十分位 → D9+D10 贡献了 65% 营收
   → 结论：头部集中，少数产品/渠道贡献大头

3. 聚类 → 识别出 3 个产品群：高营收低成本、高营收高成本、低营收高成本
   → 行动：砍掉"低营收高成本"产品线，资源倾斜"高营收低成本"

综合结论：
  - 旺季（Q4）集中推广高营收低成本产品
  - 淡季（Q1）维持基本盘，控制成本
```

## 五、图表交互操作

| 功能 | 操作方式 |
|------|---------|
| 切换指标 | 点击指标按钮（如 revenue → cost） |
| 切换周期 | 时序分析：月/季/年 按钮 |
| 切换坐标轴 | 聚类分析：X轴/Y轴下拉框 |
| 调整 K 值 | 聚类分析：在配置面板修改 |
| 查看详情 | 时序分析：滚动下方数据表 |
| 缩放 | 鼠标滚轮或拖拽 dataZoom 滑块 |
