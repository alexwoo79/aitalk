# 数据定义流转说明

## 概述

「数据定义」是 SmartBoard 中对数据列进行精细化配置的系统，包括列格式（百分比/货币/整数）、列颜色（背景色/字体色）、条件着色规则、计算列等。本文档说明这些定义如何在系统中创建、保存、重置、预览和最终应用到看板。

## 二、核心数据结构

### TableConfig（config-store 中的定义）

```ts
// src/types/config.ts
interface TableConfig {
  columns: string[]           // 显示的列
  sortBy?: string             // 排序列
  summaryAggs?: Record<...>   // 底部汇总行
  columnColors?: Record<string, string>         // 列背景色
  columnTextColors?: Record<string, string>     // 列字体色
  columnTextRules?: Record<string, ColumnTextRule[]>  // 条件着色
  rowConditionColors?: RowConditionColor[]      // 行条件颜色
  columnOrder?: string[]                        // 列显示顺序（拖拽排序）
  computedColumns?: ComputedColumn[]            // 计算列
  columnFormats?: Record<string, {              // 列格式
    format?: 'number' | 'integer' | 'percent' | 'currency'
    unit?: 'yuan' | 'wan' | 'yi'
    prefix?: string
    decimals?: number
  }>
}
```

### TableSpec（Dashboard 渲染使用）

```ts
// src/types/spec.ts — buildSpec() 的输出
interface TableSpec {
  columns: string[]
  sortBy: string
  columnColors?: Record<string, string>
  columnTextColors?: Record<string, string>
  columnTextRules?: Record<string, ColumnTextRule[]>
  columnFormats?: Record<string, { format?, unit?, prefix?, decimals? }>
  // ...
}
```

> **注意**：`buildSpec()` 仅在 `localStorage['smartboard-advanced-config'] === '1'` 时将 `columnFormats` 传入 spec。用户必须至少一次开启过「高级数据定义」开关。

## 三、编辑 → 预览的即时生效链路

### 3.1 编辑入口

```
ColumnConfigPanel (setColFormatField)
        │
        │  修改 configStore.config.table.columnFormats = { ...cf }
        │  （创建新对象引用以触发 Vue 响应式）
        ▼
configStore.config.table.columnFormats 已更新
```

### 3.2 预览表响应

**DataPreview.vue — 样本数据表**（上传页）
```ts
// 基础表 & 高级表均使用同一函数
function formatPreviewValue(col: string, val: any): string {
  const fmt = configStore.config.table.columnFormats?.[col]
  if (fmt?.format) {
    const n = getNumericVal(val)  // ← 关键：处理 CSV 字符串数字
    if (!isNaN(n)) {
      // 按 format 类型格式化
    }
  }
  return truncate(val)
}
```

**ConfigView.vue — 右侧预览面板数据表**
```ts
function previewCellValue(val: any, col?: string): string {
  const n = getNumericVal(val)
  if (!isNaN(n)) {
    const fmt = configStore.config.table.columnFormats?.[col]
    if (fmt?.format) { /* 格式化 */ }
    return n.toLocaleString(...)
  }
  return String(val)
}
```

**RelationConfig.vue — 合并表预览**
```ts
function formatMergedCell(col: string, val: any): string {
  const fmt = configStore.config.table.columnFormats?.[col]
  if (fmt?.format) {
    const n = getNumericVal(val)
    if (!isNaN(n)) return fmtByChart(n, ...)
  }
  return truncateVal(val)
}
```

### 3.3 关键注意事项

| 问题 | 原因 | 修复方式 |
|------|------|----------|
| 格式不生效 | CSV 数据值是字符串 `"1234"` 而非数字 `1234` | 使用 `getNumericVal()` 替代 `typeof val === 'number'` 判断 |
| 条件着色不生效 | 同上 | `new Function` 的参数也需 `getNumericVal` 解析 |

## 四、保存（💾 按钮）的生命周期

### 4.1 保存 ≠ 写入

**格式修改在编辑时已实时生效**。保存按钮的作用是**锁定快照**：

```
saveSection('table')
  │
  ├── 未保存 → 保存：记录当前 config.value.table 的快照
  │   sectionSnapshots['table'] = JSON.parse(JSON.stringify(config.value.table))
  │   按钮显示 ✅
  │
  └── 已保存 → 解除：删除快照
      delete sectionSnapshots['table']
      按钮显示 💾
```

### 4.2 各页面保存入口

| 页面 | 按钮 | 行为 |
|------|------|------|
| DataPreview 高级定义区 | 💾 / ✅ | `saveSection('table')` |
| RelationConfig 合并表定义区 | 💾 / ✅ | `saveSection('table')` |
| ConfigView 顶部 | 全部保存 | `saveAll()` → 批量保存 7 个区域 |

### 4.3 状态判定

```ts
function isSectionSaved(section): boolean {
  const snap = sectionSnapshots[section]     // 上次保存的快照
  if (!snap) return false                     // 从未保存
  return JSON.stringify(currentValue) === JSON.stringify(snap)  // 当前值与快照一致
}
```

## 五、重置（↺ 按钮）的完整链路

### 5.1 重置入口

| 入口 | 按钮位置 | 调用 | 影响范围 |
|------|----------|------|----------|
| DataPreview ↺ | 高级数据定义标题栏 | `resetSectionToAuto('table')` | 仅 table 区域 |
| RelationConfig ↺ | 合并表定义标题栏 | `resetSectionToAuto('table')` | 仅 table 区域 |
| ConfigView「重置全部（含列定义）」 | 顶部下拉菜单 | `resetAllToAuto()` | **全部 7 个区域** |
| ConfigView「重置看板配置（保留列定义）」 | 顶部下拉菜单 | `resetDashboardToAuto()` | title/filters/kpis/charts（**跳过 table**） |

### 5.2 resetAllToAuto 完整调用链

```
resetAllToAuto()
  │
  ├── sectionSnapshots = {}            // 清空所有快照
  │
  └── generateAutoConfig()
        │
        ├── generateAutoTitle()        // 标题 → 自动
        ├── generateAutoFilters()      // 筛选 → 自动
        ├── generateAutoKpis()         // KPI → 自动
        ├── generateAutoCharts()       // 图表 → 自动
        ├── generateAutoTable()        // 数据表 → 自动 ★
        └── generateAutoLayout()       // 布局 → 自动
```

### 5.3 generateAutoTable() 重置的属性明细

```ts
config.value.table = {
  sortBy:          autoSelected,       // 主指标列或首个指标列
  columns:         autoFiltered,       // 排除已移除列的所有列
  summaryAggs:     autoGenerated,      // 数值列→sum, 维度列→unique_count
  computedColumns: maybeCC,            // 保留用户计算列 + 预设占比列
  // ── 以下全部归零 ──
  columnFormats:        {},            // ✅ 已修复（之前保留了旧值）
  columnColors:         undefined,
  columnTextColors:     undefined,
  columnTextRules:      undefined,
  rowConditionColors:   undefined,
  columnOrder:          undefined,
}
```

### 5.4 resetDashboardToAuto（保留列定义）

```
resetDashboardToAuto()
  │
  ├── generateAutoTitle()      // 重置
  ├── generateAutoFilters()    // 重置
  ├── resetDateColumn()        // 重置
  ├── generateAutoKpis()       // 重置
  └── generateAutoCharts()     // 重置
  （不调用 generateAutoTable）  // ← 保留列格式/颜色/计算列
```

## 六、Dashboard 生成链路

### 6.1 buildSpec() 如何读取列定义

```ts
// src/stores/preview-store.ts
function buildSpec(): DashboardSpec | null {
  const cfg = configStore.config
  const advEnabled = localStorage.getItem('smartboard-advanced-config') === '1'

  // KPI 格式：优先用 columnFormats 覆盖 KPI 自身的 global/number 格式
  const defs = advEnabled ? (cfg.table.columnFormats || {}) : {}
  // ...

  // Table 格式：仅在 advEnabled 时传入
  columnFormats: advEnabled ? cfg.table.columnFormats : undefined
}
```

### 6.2 Dashboard 表格格式化

```ts
// src/views/DashboardView.vue
function formatCellValue(val, col): string {
  const cls = getEffectiveClassification(col)
  if (cls?.type === 'numeric' && cls.role === 'metric') {
    const n = getNumericVal(val)          // ← 参考实现
    if (!isNaN(n)) {
      const def = configStore.config.table.columnFormats?.[col]
      if (def?.format) return fmtByChart(n, ...)
      return fmt(n, 2)
    }
  }
  return String(val)
}
```

## 七、跨页面数据一致性

```
                 ┌──────────────────┐
                 │   configStore    │  (Pinia 全局单例)
                 │   .config.table  │
                 └────────┬─────────┘
                          │
        ┌─────────────────┼─────────────────┐
        ▼                 ▼                  ▼
  DataPreview      RelationConfig      ConfigView
  (上传页)          (关联页)            (配置页)
        │                 │                  │
        │  共享同一       │   共享同一        │   共享同一
        │  ColumnConfig   │   ColumnConfig   │   configStore
        │  Panel 组件     │   Panel 组件     │
        ▼                 ▼                  ▼
  格式修改即时          格式修改即时        预览表即时
  反映在样本表          反映在合并表        反映在右侧面板
```

> **关键**：三个页面共享同一个 `configStore.config.table`。在上传页修改格式，配置页预览表会同步更新。重置操作同样影响全局。

## 八、常见问题排查

| 现象 | 原因 | 解决 |
|------|------|------|
| 格式设置后预览表不变 | 数据值为字符串，`typeof` 检查失败 | 用 `getNumericVal()` 解析 |
| 重置后格式仍在 | `generateAutoTable()` 保留了旧 `columnFormats` | 已修复，现清空为 `{}` |
| Dashboard 不显示格式 | `advEnabled` 为 false | 在上传页开启「高级数据定义」开关 |
| 条件着色规则不生效 | 传递给 `new Function` 的是字符串 | 传入前用 `getNumericVal()` 解析 |
