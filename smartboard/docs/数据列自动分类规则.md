# 数据列自动分类规则 — 培训手册

> 代码位置：`src/core/classifier.ts`

---

## 一、分类流程总览

```
CSV/XLSX 导入
    │
    ▼
┌─────────────────────────────────┐
│ 1. 预处理：过滤空列（>95%空→忽略）│
│ 2. 采样：最多取 500 条加速       │
│ 3. 检测：日期 / 数值 / 唯一性     │
│ 4. 决策树：分派 type + role      │
│ 5. 后处理：选主指标 + 图表维度    │
└─────────────────────────────────┘
    │
    ▼
每一列得到一个分类标签
```

---

## 二、三个核心检测指标

对每一列的采样值（最多500条），计算三个比率：

| 指标 | 计算方式 | 用途 |
|------|---------|------|
| **日期比率** `dateRatio` | 匹配日期格式的值 / 采样数 | 判断是否日期列 |
| **数值比率** `numericRatio` | 可解析为数字的值 / 采样数 | 判断是否数值列 |
| **唯一比率** `uniqueRatio` | 去重后个数 / 采样数 | 判断离散程度 |

### 日期识别（4种格式）

```
2024-01-15      ← YYYY-MM-DD
2024/1/15       ← YYYY/M/D
1-15-2024       ← M-D-YYYY
2024年1月15日    ← 中文日期
20240115        ← 8位纯数字
```

### 数值解析规则

- 自动去除千分位逗号 `,` 和百分号 `%`
- 空字符串、单独的 `-` 不算数值
- 大整数（如 100000）仍然是数值

### ID 列识别（列名关键词）

```
序号 | id | 编号 | no | code | index | num | 号码 | 编码 | 工号
```

## 三、决策树（5级分类）

```
                    ┌─────────────┐
                    │  一列数据    │
                    └──────┬──────┘
                           │
                    ┌──────▼──────┐
                    │ dateRatio   │
                    │   > 80% ?   │──是──▶ 🗓 date / time_axis（时间轴）
                    └──────┬──────┘
                           │ 否
                    ┌──────▼──────┐
                    │ numericRatio│
                    │   > 80% ?   │──是──▶ 🔢 numeric → 子分类 ↓
                    └──────┬──────┘
                           │ 否
                    ┌──────▼──────┐
                    │ uniqueCount │
                    │ ≤ 20 且     │──是──▶ 📊 categorical / dimension（维度）
                    │ ≤ 50%总量 ? │
                    └──────┬──────┘
                           │ 否
                    ┌──────▼──────┐
                    │ uniqueRatio │
                    │   > 90% ?   │──是──▶ 🏷 text / label（标签）
                    └──────┬──────┘
                           │ 否
                           ▼
                    🏷 text / label（标签-兜底）
```

### 数值列子分类（numericRatio > 80% 时）

```
数值列
  ├─ 整数 + 唯一值<15 + 均值<1000  → 📊 dimension（维度）
  │   例: 评分1-5, 等级1-10
  │
  ├─ 整数 + (唯一率>85% 或 ID列名) + 均值<1000 → 🏷 label（标签）
  │   例: 序号, ID, 编号
  │
  └─ 默认 → 📈 metric（指标）
       ├─ 列名含"率/ratio/rate/percent" → format: percent
       ├─ 值含 % 符号                    → format: percent
       ├─ 全为整数                       → format: integer
       └─ 其他                          → format: number
```

## 四、5 种 Role（角色）含义

| Role | 含义 | 出现在哪里 |
|------|------|-----------|
| `time_axis` | 时间轴 | 日期筛选、时序图表 X 轴 |
| `dimension` | 维度 | 筛选下拉框、图表分组依据 |
| `metric` | 指标 | KPI 卡片、图表 Y 轴数值 |
| `label` | 标签 | 数据表的行标识，不参与图表 |
| `ignore` | 忽略 | 超过 95% 为空，完全排除 |

## 六、主指标自动选择

系统从所有 `metric` 列中**打分选出一个主指标**：

| 加分条件 | 分值 |
|---------|------|
| 非零值存在 | +1 |
| 非零值占比 > 50% | +1 |
| 唯一值 > 3 | +1 |
| 均值绝对值 > 1 | +1 |
| format 是 currency/number | +1 |
| **列名含业务关键词** | **+2** |

### 业务关键词（命中 +2 分）

```
中文: 额 | 金额 | 收入 | 利润 | 销售 | 合同 | 订单 | 数量
英文: amount | revenue | sales
```

> 当前数据中 `revenue` 命中 "revenue" 关键词，得分最高 → 被选为主指标。

## 八、分类规则对 Dashboard 的影响

| 分类结果 | 影响 |
|---------|------|
| `dimension` 列 | 自动出现在筛选栏下拉框、图表维度选项中 |
| `metric` 列 | 自动出现在 KPI 指标选项、图表数值选项中 |
| `time_axis` 列 | 激活日期范围筛选器、时序分析可用 |
| `label` 列 | 仅显示在数据表中，不参与图表 |
| `ignore` 列 | 完全不显示 |
| `primaryMetric` | KPI 卡片和图表默认使用的指标 |
| 数值列 `> <` datalist | 只有 type=numeric 的列才提示 `>` `<` |
| 文本列 `in ~` datalist | 只有 type≠numeric 的列才提示 `in` `~` |

## 九、常见场景判定

| 数据场景 | 分类结果 |
|---------|---------|
| 员工工号 `E001, E002, ...` | numeric + 高唯一率 + ID列名 → label |
| 产品评分 `1~5` | numeric + 整数 + 低唯一 → dimension |
| 销售额 `12800, 35000, ...` | numeric + 高唯一 → metric |
| 省份 `北京, 上海, ...` | categorical + ≤20种 → dimension |
| 客户名称 `张三, 李四, ...` | text + 高唯一率 → label |
| 日期 `2024-01-01` | date + 高日期率 → time_axis |
| 全空列（>95%空） | → ignore |
