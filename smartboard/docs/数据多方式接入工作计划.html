SmartBoard 数据多方式接入 — 工作计划

日期：2026-06-27 | 状态：规划中

一、现状诊断

1.2 结论

后端并没有”先加了但前端没入口”的情况。 当前前后端能力一一对应，都仅支持 CSV/Excel 文件加载。需要前后端同步新增多种数据接入方式。

二、总体策略

分 4 个 Phase，按实现成本从低到高、用户价值从高到低排序：

Phase 方式 后端工作量 前端工作量 优先级
Phase 1 📋 剪贴板粘贴 小（新增 1 个 command） 中（新增 PasteZone 组件） ⭐⭐⭐
Phase 2 📄 JSON / Parquet 小（扩展现有 loader） 小（扩展文件过滤） ⭐⭐
Phase 3 🌐 URL / API 拉取 中（新增 reqwest 依赖） 中（新增 URL 输入组件） ⭐⭐
Phase 4 🗄️ SQLite 直连 大（新增 rusqlite 依赖） 大（新增数据库配置 UI） ⭐

Phase 1: 📋 剪贴板粘贴导入

目标：用户从 Excel/WPS/Google Sheets 复制表格数据，Ctrl+V 粘贴到 SmartBoard，自动解析为 DataFrame。

1.1 后端 (Rust)

新增 Tauri Command: paste_from_clipboard

// src-tauri/src/commands/clipboard.rs (新文件)

#[tauri::command]
pub async fn paste_from_clipboard(
    text: String,       // 剪贴板原始文本（TSV 格式）
    has_header: bool,   // 第一行是否为表头
) -> ApiResult<ChartPayload>

实现要点： - 使用 Polars CsvReader 以 \t 为分隔符解析 TSV - 支持 has_header 参数（用户可切换首行是否为表头） - 自动调用 auto_cast_numeric 做数值类型推断 - 注册为活跃数据集

依赖：无需新增 Rust 依赖（复用现有 Polars）

文件变更： - 新增 src-tauri/src/commands/clipboard.rs (~50 行) - 修改 src-tauri/src/lib.rs 注册命令

目标：扩展现有文件加载能力，支持 .json（数组或行-oriented）和 .parquet 格式。

扩展现有 load_file_impl 支持 JSON

// 在 load_file_impl 中增加格式判断
match ext {
    "csv" | "tsv" | "txt" => { /* 现有 CSV 逻辑 */ }
    "xlsx" | "xls" | "xlsm" | "ods" => { /* 现有 Excel 逻辑 */ }

#[tauri::command]
pub async fn fetch_from_url(
    url: String,            // 数据 URL
    format: String,         // "csv" | "json" | "auto"

Phase 4: 🗄️ SQLite 数据库直连

目标：用户选择本地 .db / .sqlite 文件，选择表导入。

4.1 后端 (Rust)

新增依赖

# Cargo.toml
rusqlite = { version = "0.31", features = ["bundled"] }

新增 Tauri Commands

// src-tauri/src/commands/database.rs (新文件)

/// 列出 SQLite 数据库中的所有表
#[tauri::command]
pub async fn list_sqlite_tables(
    path: String,
) -> ApiResult<Vec<TableInfo>>

/// 从 SQLite 加载指定表
#[tauri::command]
pub async fn load_sqlite_table(
    path: String,
    table_name: String,
) -> ApiResult<ChartPayload>

/// 执行自定义 SQL 查询
#[tauri::command]
pub async fn execute_sql_query(
    path: String,
    query: String,
) -> ApiResult<ChartPayload>

实现要点： - list_sqlite_tables：查询 sqlite_master 获取所有用户表 - load_sqlite_table：SELECT * FROM {table} 加载全表 - execute_sql_query：允许高级用户自定义 SQL（安全考虑：仅允许 SELECT） - 使用 bundled feature 避免系统 SQLite 版本依赖

文件变更： - 修改 src-tauri/Cargo.toml 添加 rusqlite - 新增 src-tauri/src/commands/database.rs (~120 行) - 修改 src-tauri/src/lib.rs 注册命令

七、风险与注意事项

  1. 剪贴板权限：浏览器环境需要用户授权 clipboard-read 权限，HTTPS 环境下才可用。Tauri 环境可直接通过 paste 事件获取。
  2. 大文件 URL 拉取：reqwest 默认全量下载到内存，超大文件（>100MB）需后续增加流式处理。
  3. SQL 注入：Phase 4 的自定义 SQL 功能仅允许 SELECT 语句，需在 Rust 侧做 SQL 白名单校验。
  4. JSON 嵌套展平：深层嵌套 JSON 展平策略需与用户确认（如 {a: {b: 1}} → 列名 a.b）。
