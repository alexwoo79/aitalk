# 计算列占比逻辑说明

## 概述

Smartboard 的计算列支持 `SUM()`、`AVG()` 等聚合函数。当表达式包含聚合函数（如 `A/SUM(A)`）时，系统会自动处理两层筛选的占比语义，实现类似 Power BI 的 **% of Grand Total** 与 **% of Parent** 计算。

---

## 数据流架构

```
┌─────────────────────────────────────────┐
│          顶部筛选栏（Dashboard 级别）      │
│  日期范围 / 列值筛选 / 条件表达式          │
│          ↓ 产出 filteredRows             │
├─────────────────────────────────────────┤
│  precomputeAggregates                    │
│  从 filteredRows 计算 SUM(A)            │
│  → 仪表板范围内的"全局"总和               │
│          ↓                              │
│  每行: A_i / SUM(A)_dashboard           │
├─────────────────────────────────────────┤
│          表格级筛选（搜索 / 条件）         │
│  仅控制表格中哪些行显示                   │
│  不影响 SUM(A) 的值                      │
│          ↓                              │
│  表格显示 + 合计行                        │
└─────────────────────────────────────────┘
```

---

## 两种占比模式

### 模式 1：无顶部筛选 → 全局占比

| 顶部筛选 | SUM(A) 来源 | 表格筛选 | 合计结果 |
|---------|------------|---------|---------|
| **无** | 全量数据 | 有条件 | **< 100%** |

- 每行占比 = $A_i$ / 全量数据的 SUM(A)
- 表格筛选是全局数据的子集，所以合计 < 100%
- 语义：**"这部分数据在全局中占多少"**

### 模式 2：有顶部筛选 → 子集内相对占比

| 顶部筛选 | SUM(A) 来源 | 表格筛选 | 合计结果 |
|---------|------------|---------|---------|
| **有** | 筛选后数据 | 有条件 | **= 100%** |

- 每行占比 = $A_i$ / 筛选后数据的 SUM(A)
- 表格筛选是同一子集的进一步筛选，所以合计 = 100%
- 语义：**"在这个筛选范围内，各条目占比如何"**

## 技术实现

核心代码位于 `DashboardView.vue` 的 `tableFilteredRows` 计算属性：

```typescript
const tableFilteredRows = computed(() => {
  // 第1步：获取仪表板级别的行（受顶部筛选影响）
  const rows = previewStore.filtersApplied
    ? previewStore.filteredRows    // 有筛选 → 筛选后的子集
    : previewStore.effectiveRows  // 无筛选 → 全量数据

  // 第2步：基于上面的行，预计算 SUM(A) 等聚合值
  let filtered = augmentWithComputedCols(rows)

  // 第3步：表格级别的搜索/条件筛选（仅影响显示，不影响 SUM(A)）
  if (tableSearch.value) filtered = filtered.filter(...)
  if (tableCondition.value) filtered = applyFilter(filtered, ...)

  return filtered
})
```

关键点：`precomputeAggregates` 在第 2 步使用第 1 步的 `rows` 计算 SUM(A)，而第 3 步的表格筛选发生在之后。这保证了 SUM(A) 的"分母"始终是仪表板范围，表格筛选只影响"分子"的可见性。

## 与 Power BI 的对比

| 场景 | Smartboard | Power BI |
|------|-----------|----------|
| 无全局筛选 + 表格筛选 | 子集 / 全局（<100%） | `% of Grand Total` |
| 有全局筛选 + 表格筛选 | 子集 / 子集（=100%） | `% of Parent` |

---

## 用户操作指南

1. **想看全局占比**：顶部筛选栏保持全部（默认），在表格中使用搜索/条件筛选看某类数据占全局的比例
2. **想看子集内占比**：先在顶部筛选栏设置条件（如日期范围），再在表格中筛选，占比会在子集内重新归一化到 100%

---

## 注意事项

- 纯聚合表达式（如 `SUM(A)`）会在每行显示相同的全局汇总值，建议用于中间变量，不作为最终显示列
- 浮点精度：合计值可能出现 $1.0000000000000002$ 而不是 $1.0$，系统已内置 $10^{-6}$ 容差处理
- 表格的"搜索"和"条件"筛选不会改变占比的分母，仅控制可见行
