以下是我针对**首页（上传数据源阶段）**的布局重构建议，以及**关联操作**的交互设计方案：

---

### 一、 首页布局重构：从“单文件上传”变为“数据源工作区”

目前的“拖拽上传区”占满首页，无法容纳多表。建议将首页改造成 **“左侧表列表 + 右侧工作区”** 的结构：

| 区域 | 布局建议 | 功能说明 |
| :--- | :--- | :--- |
| **顶部栏** | 保持现有 SmartBoard Logo + 语言切换，新增 **“新建数据源”** 大按钮 | 点击后弹出上传弹窗，支持批量上传（多选文件） |
| **左侧面板（表管理）** | 宽度约 240px，展示已导入的所有表格列表（如：收入台账、支出合同、客户主数据） | 每个表项带有**状态标识**（绿色圆点=已解析，灰色=待处理），并支持**重命名**、**删除**、**查看原始数据** |
| **右侧主视图（核心变化）** | 分为两个 Tab：**“表预览”** 和 **“关联关系”** | 默认展示“表预览”，点击单表显示前100行数据和列分类结果（复用你现有的列分类卡片） |

### 二、 核心难点：“建立表内连接”的交互布置

这里的“表内连接”我理解是指**多表之间的关联（Join/Relation）**。不要把它放在复杂的弹窗里，建议直接在首页右侧的 **“关联关系”Tab** 中以**可视化画布**呈现：

#### 1. 可视化关联画布（拖拽式）
- **画布展示**：将左侧已导入的所有表，以“卡片节点”形式展示在画布上（类似 ER 图）。
- **建立关联动作**：用户从一张表的某个字段（如 `合同编号`）**拖拽出一条连线**，连到另一张表的字段（如 `合同编号`）上。
- **智能辅助**：系统自动高亮匹配的数据类型（文本对文本，数值对数值），如果字段名相似（如 ID 与 ID），可弹出气泡提示“是否自动关联？”。

#### 2. 关联配置浮窗（点击连线后弹出）
当用户拖拽完成，弹出一个轻量配置浮窗，内容包含：

- **连接类型**（下拉选择）：左连接、内连接、全连接（对于非技术用户，用文字解释：“保留左表所有”、“仅保留匹配项”）。
- **基数识别**：自动检测并展示（1:1, 1:N, N:N），让用户确认。
- **生效范围**：勾选“此关联仅用于本次看板”或“设为全局关联”。

#### 3. 关联后的“逻辑主表”概念
- 在画布上方加一句提示：**“请指定分析主表”**（让用户点击其中一张表设为星标）。
- 设置主表后，**“配置看板”** 步骤（第二步）中的字段选择器里，将自动**带上表名前缀**（例如 `收入台账.合同总金额` 和 `客户主数据.客户区域`），方便用户混搭字段制作 KPI 和图表。

---

### 三、 流程调整（Step 1 变更为 4 个子状态）

建议将首页步骤条下方的区域，由单一的“上传区”改为**四个纵向步骤指示器**（但保持在同一个页面）：

1. **📤 上传文件**（批量拖拽区，保留你的“下载示例数据”和“数据质量要求”折叠）
2. **🧹 清洗配置**（沿用你现有的“列分类结果”卡片，现在支持切换不同表查看）
3. **🔗 建立关联**（**新增核心步骤**，展示上述可视化画布）
4. **✅ 完成准备**（点击后进入第二步“配置看板”）

> 如果用户只导入单表，自动跳过第 3 步（关联），直接高亮“完成准备”按钮。

---

### 四、 针对你现有界面的微调建议（锦上添花）

- **数据量提示**：在左侧表列表显示行数（如 `389行` 你已做了），多表场景下额外显示 `7列`。
- **关联状态反馈**：在“配置看板”页面（第二步），如果用户未建立关联就试图生成 Dashboard，在“生成”按钮旁显示弱提示：“检测到多张表未关联，部分字段可能无法跨表计算”。
- **性能兜底**：在多表上传区下方，增加一行小字：“多表关联时，系统将采用抽样预览（前1000行）以保障性能，生成看板时使用全量数据”。

**总结建议**：不要强行将“多表关联”塞进原来的单列卡片布局里，而是**利用左侧边栏管理表资源，利用右侧 Tab 区分“预览”和“建模”**。这种布局符合 Tableau/Power BI 的用户心智，且能让你的“列分类结果”功能（目前做得很好）无缝迁移到“表预览”Tab 下。
